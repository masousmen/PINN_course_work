# Выбранные кейсы

## Основные

- `heat_alpha01` - проверка. На простой heat-задаче обе точности работают хорошо. Это простая проверка всей схемы обучения.
- `helmholtz_m12_long` - главный положительный пример. В этом запуске есть по два валидных seed, и медиана FP64 заметно ниже медианы FP32.
- `helmholtz_m12_rs` - дополнительный чистый пример. Ещё один устойчивый m=12 с ресемплированием. FP64 снова даёт меньшую медианную ошибку.
- `helmholtz_m12_resample128` - дополнительный чистый пример. Похожая настройка с более длинным L-BFGS. Вывод совпадает с основным m=12.
- `helmholtz_m7_rs` - положительный пример. На меньшем m результат тоже чистый: оба dtype имеют по два валидных seed.
- `helmholtz_m11_rs` - умеренно положительный пример. FP64 лучше по медиане, но отрыв меньше, чем на m=12.
- `convection_beta30` - аккуратный пример convection. Есть по два seed у FP32 и FP64. FP64 лучше умеренно, без истории про полный провал FP32.
- `burgers_nu0p002` - результаты близкие. Burgers nu=0.002 показывает близкие результаты FP32 и FP64.
- `burgers_nu0p001` - смешанный результат. На этом Burgers-запуске FP64 не даёт устойчивого преимущества. Это полезный отрицательный пример.
- `fp16_summary` - FP16 нестабилен. FP16 вынесен отдельно: 22/22 запусков плохие или невалидные.

## Helmholtz

- `helmholtz_m12_long` - главный положительный пример. В этом запуске есть по два валидных seed, и медиана FP64 заметно ниже медианы FP32.
- `helmholtz_m12_rs` - дополнительный чистый пример. Ещё один устойчивый m=12 с ресемплированием. FP64 снова даёт меньшую медианную ошибку.
- `helmholtz_m12_resample128` - дополнительный чистый пример. Похожая настройка с более длинным L-BFGS. Вывод совпадает с основным m=12.
- `helmholtz_m7_rs` - положительный пример. На меньшем m результат тоже чистый: оба dtype имеют по два валидных seed.
- `helmholtz_m8_rs` - положительный пример. Это не старый unstable m=8, а более аккуратный rs-запуск без плохих seed.
- `helmholtz_m11_rs` - умеренно положительный пример. FP64 лучше по медиане, но отрыв меньше, чем на m=12.
- `helmholtz_m10_resample128` - умеренный пример. Можно использовать как вспомогательный пример: FP64 лучше не так резко.
- `helmholtz_m10_initial` - умеренный пример. Старый запуск m=10 полезен как дополнительная проверка, но не главный результат.

## Диагностические

- `convection_beta50` - требует проверки. FP64 выглядит сильно лучше, но здесь только один seed. FP32 мог не сойтись из-за неудачного старта, поэтому кейс нельзя делать главным выводом.
- `helmholtz_m8_main_old` - зависит от seed. Старый m=8 показывает сильную зависимость от seed. Его полезно оставить как пример нестабильности.
- `helmholtz_m15_helmholtz_m15` - сложный режим. При большем m обе точности часто не сходились. Это уже скорее граница выбранной схемы обучения.
